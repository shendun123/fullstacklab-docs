# 接口文档

全栈实验室 支持 OpenAI、Anthropic 和 Gemini 三类协议。统一地址：

```text
https://api.fullstacklab.xyz
```

### OpenAI 协议

基础信息：

```text
API 端点: https://api.fullstacklab.xyz/v1
认证方式: Bearer Token
请求头: Authorization: Bearer YOUR_API_KEY
```

聊天补全示例：

```bash
curl https://api.fullstacklab.xyz/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "gpt-4o",
    "messages": [
      {"role": "user", "content": "Hello"}
    ]
  }'
```

常用参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| model | string | 是 | 模型名称 |
| messages | array | 是 | 消息列表 |
| temperature | number | 否 | 采样温度 |
| max_tokens | integer | 否 | 最大生成 token 数 |
| stream | boolean | 否 | 是否流式输出 |

Responses API 端点：

```text
POST /v1/responses
```

常用字段包括 `model`、`input`、`instructions`、`max_output_tokens`、`temperature`、`top_p`、`stream`、`tools`、`tool_choice`、`reasoning`、`previous_response_id` 和 `truncation`。

示例：

```bash
curl -X POST "https://api.fullstacklab.xyz/v1/responses" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4o",
    "input": "Hello"
  }'
```

### Anthropic 协议

基础信息：

```text
API 端点: https://api.fullstacklab.xyz/v1/messages
认证方式: Bearer Token 或 X-API-Key
必要请求头: anthropic-version: 2023-06-01
```

示例：

```bash
curl https://api.fullstacklab.xyz/v1/messages \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "anthropic-version: 2023-06-01" \
  -d '{
    "model": "claude-sonnet-4-20250514",
    "max_tokens": 1024,
    "messages": [
      {"role": "user", "content": "Hello"}
    ]
  }'
```

常用参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| model | string | 是 | 模型名称 |
| max_tokens | integer | 是 | 最大输出 token 数 |
| messages | array | 是 | 消息列表 |
| system | string | 否 | 系统提示词 |
| temperature | number | 否 | 温度 |
| stream | boolean | 否 | 是否流式输出 |

### Gemini 协议

基础信息：

```text
API 端点: https://api.fullstacklab.xyz/v1beta/models
认证方式: URL 参数或 Bearer Token
请求头: Content-Type: application/json
```

示例：

```bash
curl https://api.fullstacklab.xyz/v1beta/models/gemini-2.5-pro:generateContent \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "contents": [
      {
        "parts": [
          {"text": "Hello"}
        ]
      }
    ],
    "generationConfig": {
      "temperature": 0.7,
      "maxOutputTokens": 1024
    }
  }'
```

常用参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| contents | array | 是 | 内容列表 |
| contents[].parts[].text | string | 是 | 文本内容 |
| generationConfig.temperature | number | 否 | 温度 |
| generationConfig.maxOutputTokens | number | 否 | 最大输出 token 数 |
| systemInstruction | object | 否 | 系统指令 |

### 流式输出

OpenAI 和 Anthropic 通常通过请求体里的 `stream: true` 开启流式响应。Gemini 可使用 `streamGenerateContent` 端点。

### 错误响应

常见错误格式：

```json
{
  "error": {
    "code": "invalid_request_error",
    "message": "Invalid API key provided",
    "status": 401
  }
}
```

常见状态码：

| 状态码 | 说明 |
| --- | --- |
| 400 | 请求参数错误 |
| 401 | 认证失败 |
| 403 | 权限不足 |
| 429 | 请求频率超限 |
| 500 | 服务端错误 |

### SDK 示例

OpenAI SDK：

```python
from openai import OpenAI

client = OpenAI(
    api_key="YOUR_API_KEY",
    base_url="https://api.fullstacklab.xyz/v1"
)

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "Hello"}]
)

print(response.choices[0].message.content)
```

Anthropic SDK：

```python
from anthropic import Anthropic

client = Anthropic(
    api_key="YOUR_API_KEY",
    base_url="https://api.fullstacklab.xyz/v1"
)

response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    messages=[{"role": "user", "content": "Hello"}]
)

print(response.content[0].text)
```

Gemini SDK 连接兼容服务时，通常需要根据具体 SDK 对自定义 base URL 的支持情况调整配置。若 SDK 不方便设置代理地址，可以优先使用 REST 请求方式。
