# 接口文档

全栈实验室接口文档已按具体请求类型拆分，方便单独查看和复制使用。

统一基础地址：

```text
https://fullstacklab.xyz
```

OpenAI 兼容协议通常使用：

```text
https://fullstacklab.xyz/v1
```

## 模型与辅助接口

### 列出模型

用于查看当前 Key 可用模型：

```bash
curl "https://fullstacklab.xyz/v1/models" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

返回结果通常包含：

```text
data[].id
data[].owned_by
```

写文档或调试时，模型名称优先以后台和 `/v1/models` 返回值为准。

### Embeddings

```text
POST https://fullstacklab.xyz/v1/embeddings
```

适合文本向量化、检索、聚类、推荐等场景。示例模型以后台可用模型为准，Apifox 示例中使用 `text-embedding-ada-002`。

### Web Search / Reader

Apifox 文档中还包含 Web 搜索和网页阅读接口：

```text
POST https://fullstacklab.xyz/v1/web-search/search
POST https://fullstacklab.xyz/v1/web-reader/read
```

`web-search/search` 适合搜索查询，常用参数包括 `query`、`area`、`freshness`、`includeImages`、`page`、`count`、`format`。  
`web-reader/read` 适合读取网页内容，常用参数包括 `url`、`format`、`liteMode`、`includeImages`、`topic`。

## 使用建议

1. 先确认后台已经创建 API Key。
2. 复制接口地址时注意不要多空格，不要漏 `/v1`。
3. 测试时先用最小请求，例如简单文本、单张图片、短视频任务。
4. 图像和视频任务可能耗时较长，建议设置合理超时和轮询间隔。
5. 不要把真实 API Key 写入 GitHub、截图、公开文档或聊天记录。
