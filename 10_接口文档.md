## 基础地址

```text
https://fullstacklab.xyz
```


## 鉴权

```http
Authorization: Bearer YOUR_API_KEY
Content-Type: application/json
```

请将 `YOUR_API_KEY` 替换为后台创建的真实 Key，不要将 Key 写入公开仓库、截图或群聊。

## 查询可用模型

```bash
curl "https://fullstacklab.xyz/v1/models" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

模型名称和权限以当前 Key 返回的 `data[].id` 为准。Seedance 等模型可能包含渠道前缀或规格后缀，必须原样传入。

## 公共任务流程

图像和视频异步接口通常使用：

```text
POST /v1/videos
GET  /v1/videos/{task_id}
```

状态包括：

| 状态 | 说明 |
| --- | --- |
| queued | 排队中 |
| processing / in_progress | 处理中 |
| completed | 已完成，可以读取结果地址 |
| failed | 失败，读取 `error`、`code` 或 `message` |

结果地址可能位于 `url`、`image_url`、`video_url`、`result_url` 或 `metadata.result_url`。图片建议每 3 至 5 秒轮询，视频建议每 5 至 10 秒轮询。

## Shell、Python、Java 公共模板

以下示例使用模型列表接口测试 Key 和基础地址。

### Shell

```bash
curl "https://fullstacklab.xyz/v1/models" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Python

```python
import requests

response = requests.get(
    "https://fullstacklab.xyz/v1/models",
    headers={"Authorization": "Bearer YOUR_API_KEY"},
    timeout=60,
)
response.raise_for_status()
print(response.json())
```

### Java

```java
import java.net.URI;
import java.net.http.HttpClient;
import java.net.http.HttpRequest;
import java.net.http.HttpResponse;

public class ListModelsExample {
    public static void main(String[] args) throws Exception {
        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create("https://fullstacklab.xyz/v1/models"))
            .header("Authorization", "Bearer YOUR_API_KEY")
            .GET()
            .build();

        HttpResponse<String> response = HttpClient.newHttpClient()
            .send(request, HttpResponse.BodyHandlers.ofString());
        System.out.println(response.body());
    }
}
```

## 辅助接口

```text
POST https://fullstacklab.xyz/v1/embeddings
POST https://fullstacklab.xyz/v1/web-search/search
POST https://fullstacklab.xyz/v1/web-reader/read
```

Embeddings 示例模型为 `text-embedding-ada-002`，实际以模型列表为准。Web Search 常用参数包括 `query`、`freshness`、`page`、`count`；Web Reader 常用参数包括 `url`、`format`、`liteMode`。

## 错误与排查

| HTTP 状态 | 说明 | 处理建议 |
| --- | --- | --- |
| 400 | 参数、JSON、模型或素材错误 | 检查字段层级、必填参数和模型名 |
| 401 | Key 缺失或无效 | 检查 Bearer 鉴权 |
| 403 | 权限或额度不足 | 检查模型权限、账号分组和余额 |
| 404 | 路径或任务不存在 | 检查接口地址和任务 ID |
| 413 | 请求或素材过大 | 压缩文件、减少参考素材 |
| 429 | 请求过于频繁 | 降低并发并退避重试 |
| 500 | 服务端异常 | 保留完整响应并稍后重试 |
| 502 / 503 / 504 | 上游或网关异常 | 延迟重试、切换模型 |
| 522 / 524 | 网络连接或超时 | 检查网络并提高客户端超时 |

排查顺序：

1. 使用 `GET /v1/models` 确认模型名和权限。
2. 打印 HTTP 状态码和完整响应。
3. 确认参考素材是无需登录的公网文件直链。
4. 检查 JSON 与 multipart 字段是否混用。
5. 只在 `completed` 状态读取结果地址。
6. 视频创建超时建议设置为 300 秒，轮询总超时不少于 600 秒。
