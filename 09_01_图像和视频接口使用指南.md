# 图像和视频接口使用指南

本页根据三个测试脚本整理：GPT Image 2 同步生图、Nano Banana 任务式生图、Veo Fast 任务式视频生成。

## 三类接口的区别

| 类型 | 接口形式 | 典型模型 | 返回方式 |
| --- | --- | --- | --- |
| 同步图像生成 | `POST /v1/images/generations` | gpt-image-2 | 直接返回 `url` 或 `b64_json` |
| 任务式图像生成 | `POST /v1/videos` + `GET /v1/videos/{task_id}` | nano_banana 系列 | 先返回任务 ID，再轮询图片地址 |
| 任务式视频生成 | `POST /v1/videos` + `GET /v1/videos/{task_id}` | veo3.1-fast | 先返回任务 ID，再轮询视频地址 |

## 推荐测试顺序

1. 先用文本接口测试 API Key 是否有效。
2. 再用 GPT Image 2 做单张同步生图。
3. 然后用 Nano Banana 做单张任务式生图。
4. 最后再测试 Veo Fast 视频生成。
5. 全量测试、批量测试和高分辨率测试会产生更多计费，确认单次成功后再执行。

## 任务式接口通用流程

```text
1. POST 创建任务
2. 从返回中读取 task_id 或 id
3. GET /v1/videos/{task_id} 轮询状态
4. status=completed 时读取 url / output / video_url
5. 下载图片或视频
```

## 重要注意事项

- `API_KEY` 必须替换为用户自己在全栈实验室后台创建的真实 Key。
- 文档、GitHub、截图中不要出现真实 Key。
- 图片和视频接口耗时较长，建议设置较长 timeout。
- 图生图的参考图必须是公网可访问的图片直链。
- 视频任务建议轮询间隔 5 秒，总超时 600 秒左右。
