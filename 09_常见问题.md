# 常见问题

遇到问题时，先做三步快速排查：

1. 重新从后台复制 API Key，确认没有多余空格，也没有复制旧 Key。
2. 确认 Base URL 为 `https://fullstacklab.xyz`，并检查当前网络是否可访问。
3. 完全关闭 Claude Code、Codex 和终端后重新打开。

配置不生效常见原因：

| 现象 | 可能原因 | 处理方式 |
| --- | --- | --- |
| 保存后仍走默认配置 | 工具没有完全退出 | 关闭终端和后台进程后重开 |
| Claude Code 认证失败 | Key 或字段写错 | 检查 `ANTHROPIC_API_KEY` 和 `ANTHROPIC_BASE_URL` |
| Codex 认证失败 | `auth.json` 路径错或 Key 错 | 检查用户目录下的 `.codex/auth.json` |
| 连接失败 | Base URL 错或网络异常 | 确认地址和后台说明一致 |
| 修改没变化 | 改错配置文件 | 回到对应手动配置章节核对路径 |

关于模型和费用：

- 工具可能在标题、摘要等辅助任务中使用轻量模型。
- 缓存用于降低重复上下文成本，首次写入和后续读取可能产生不同计费。
- 对话过长触发压缩或空闲过久，都可能导致缓存变化。
- 具体费用以 全栈实验室 后台为准。

关于服务可用性：

- 全栈实验室 依赖上游官方 API。
- 上游故障、限制或策略变化可能导致多个渠道同时不可用。
- 可以先等待几分钟重试，再查看后台公告或服务通知。

如果仍无法解决，求助时建议提供：

- 操作系统
- 使用工具
- 配置方式
- 报错截图或终端最后 20 行输出
- 是否确认 Base URL

## 图像和视频接口常见问题

### 1. 为什么同步生图接口有时超时？

图片生成可能需要较长时间。建议把请求超时从 30 秒提高到 60 秒、120 秒或 300 秒，并在失败后间隔几秒重试。

### 2. 为什么返回里有时是 `url`，有时是 `b64_json`？

不同图像模型或通道返回格式可能不同。保存图片时建议同时兼容：

```text
url
b64_json
```

### 3. 为什么 Nano Banana 创建任务后没有立即返回图片？

Nano Banana 是任务式接口，创建任务后会返回 `task_id` 或 `id`，需要继续轮询：

```text
GET /v1/videos/{task_id}
```

等状态变成 `completed` 后再读取图片地址。

### 4. 为什么图生图失败？

常见原因是参考图 URL 不是公网直链，或者返回的 `Content-Type` 不是 `image/*`。建议先在浏览器里打开图片直链测试。

### 5. 为什么 Veo 视频生成要等很久？

视频生成耗时通常比图片更长。建议轮询间隔设置为 5 秒，总超时设置为 600 秒左右。

### 6. 完成后为什么没有视频地址？

不同通道返回字段可能不同，建议同时兼容：

```text
url
output
video_url
```

### 7. 为什么 401 Unauthorized？

通常是 API Key 错误、Key 未复制完整、环境变量没有生效，或者代码里仍然使用示例 Key。请重新从后台复制真实 Key，并重启终端。

## 上下文与压缩机制

上下文窗口是模型一次请求能够处理的历史消息、文件内容、工具结果等信息容量。不同模型的上下文大小不同，也可能随版本和服务能力变化。

当对话内容接近上下文上限时，Claude Code 可能会触发自动压缩。压缩会把早期内容概括成更短摘要，让任务继续进行，但也可能导致早期细节变模糊、缓存失效、费用变化或需要重新补充背景。

Claude Code 可以通过环境变量调整压缩触发阈值：

```text
CLAUDE_AUTOCOMPACT_PCT_OVERRIDE
```

常见建议值：

| 值 | 效果 | 适合场景 |
| --- | --- | --- |
| 100 | 更晚压缩 | 短对话、轻量使用 |
| 78/80 | 提前留余量 | 大多数长任务用户 |
| 60 | 更频繁压缩 | 极长任务，但更容易丢细节 |

可以放进 `~/.claude/settings.json` 的 `env` 中：

```json
{
  "env": {
    "ANTHROPIC_API_KEY": "sk-xxxxxxxxxxxxxxxx",
    "ANTHROPIC_BASE_URL": "https://fullstacklab.xyz",
    "CLAUDE_AUTOCOMPACT_PCT_OVERRIDE": "78"
  }
}
```

长期稳定的信息建议写入记忆文件，而不是只留在聊天记录里。Claude Code 使用 `CLAUDE.md`，Codex 使用 `AGENTS.md`。

## 记忆文件说明

记忆文件适合保存项目长期规则、常用命令、目录说明和业务约束。它能在工具启动或读取项目时重新进入上下文，降低长对话压缩后遗忘关键规则的概率。

Claude Code 使用 `CLAUDE.md`：

| 层级 | 路径 | 作用 |
| --- | --- | --- |
| 全局 | `~/.claude/CLAUDE.md` | 所有项目生效 |
| 项目 | `<项目根目录>/CLAUDE.md` | 当前项目生效 |
| 子目录 | `<项目子目录>/CLAUDE.md` | 当前子目录范围生效 |

Codex 使用 `AGENTS.md`：

| 层级 | 路径 | 作用 |
| --- | --- | --- |
| 全局 | `~/.codex/AGENTS.md` | 所有项目生效 |
| 项目 | `<项目根目录>/AGENTS.md` | 当前项目生效 |
| 子目录 | `<项目子目录>/AGENTS.md` | 当前子目录范围生效 |

适合写入：

- 项目技术栈
- 目录说明
- 常用开发、构建、测试命令
- 重要业务约束
- 代码风格
- 提交规范

不适合写入：

- API Key
- Cookie
- Token
- 大段聊天记录
- 过期临时问题
- 和项目无关的个人偏好

示例：

```markdown
# 项目说明

这是一个文档站项目，用来维护用户使用手册。

## 常用命令

- 安装依赖：npm install
- 本地开发：npm run dev
- 构建检查：npm run build

## 写作规则

- 面向新手用户，先给操作步骤，再解释原因。
- 不把 API Key、Cookie、Token 写进文档。
- 截图统一放在 docs/public/screenshots/。
```

记忆文件也会占用上下文，所以只写稳定、必要、长期复用的信息。

## 知识时间测试说明

不建议只用“你的知识截止到什么时候？”或“你是什么模型 ID？”来判断模型是否可靠。这类问题可以随手观察，但不适合作为准确测试。

模型不一定能知道当前服务真实设置的知识截止时间，也未必能读取实际运行时模型 ID。客户端显示名、服务端别名、分组路由和真实上游模型名可能并不一致。

更好的测试方式：

| 测试目标 | 推荐做法 | 观察重点 |
| --- | --- | --- |
| 知识可靠性 | 准备 5 到 10 个有标准答案的问题 | 是否准确，是否承认不确定 |
| 时效性 | 选择明确发生在某个时间点前后的事实 | 是否混淆时间线 |
| 代码能力 | 给一个小 bug 或小功能 | 是否能读上下文、改对文件、通过测试 |
| 长上下文能力 | 给较长需求和约束 | 是否遵守关键约束 |
| 稳定性 | 同一任务重复测试几次 | 答案质量是否大幅波动 |
| 配置是否生效 | 看客户端配置、服务端日志和用量记录 | 请求是否走目标 API 和模型 |

判断幻觉时，可以观察模型是否会在不知道时明确说不确定，是否说明依据，是否把猜测说成事实，是否编造配置项、命令、路径或版本号。
