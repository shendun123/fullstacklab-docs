# Memory Files

Memory files store long-term project rules, commands, folder descriptions, and workflow requirements.

Common memory files:

```text
CLAUDE.md
AGENTS.md
```

## What to Store

Good examples:

- Project structure
- Common commands
- Coding style
- API conventions
- Business rules
- Deployment notes

Do not store:

- API Keys
- Cookies
- Login tokens
- Private passwords
- Real customer secrets

## Usage Advice

Keep memory files short, clear, and stable. Remove outdated rules when the project changes.
